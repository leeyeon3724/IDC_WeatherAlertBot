## 1) 서비스 개요 (WthrWrnInfoService)

- 서비스 베이스 URL: `http://apis.data.go.kr/1360000/WthrWrnInfoService`
- 인터페이스: REST(GET)
- 응답 포맷: XML/JSON (`dataType`, 기본 XML)
- 인증: `serviceKey` 필수(URL Encode)

- 데이터 갱신주기: 수시(특보 발생/해제 시)

## 2) 공통 요청/응답 규칙

- 공통 요청 파라미터(대부분):
  - `serviceKey`(필수), `numOfRows`(필수, 기본 10), `pageNo`(필수, 기본 1), `dataType`(옵션, 기본 XML)
  - 일부 기능은 `stnId`, `fromTmFc`, `toTmFc`, `areaCode`, `warningType` 추가
- 공통 응답 메타: `numOfRows`, `pageNo`, `totalCount`, `resultCode`, `resultMsg`, `dataType`
- 문서의 항목구분 표기: `1=필수`, `0=옵션`, `1..n=복수 필수`, `0..n=복수 옵션`
- 시각 필드 포맷: `yyyyMMdd` 또는 `yyyyMMddHHmm`

### 코드값 기준 (문서 명세)

- `warningType` / `warnVar`: `1=강풍`, `2=호우`, `3=한파`, `4=건조`, `5=폭풍해일`, `6=풍랑`, `7=태풍`, `8=대설`, `9=황사`, `12=폭염`
- `warnStress`: `0=주의보`, `1=경보`
- `command`: `1=발표`, `2=해제`, `3=연장`, `6=정정`, `7=변경발표`, `8=변경해제`
- `cancel`: `0=정상`, `1=취소된 특보`

### 문서 내 주의사항(실무 반영 필요)

- `getPwnCd` 요청 예제 URL에는 `warninType` 오탈자가 있으나, 요청 항목 표는 `warningType`입니다.
- `resultCode`는 항목 샘플에 `00`, 예제 XML에는 `0`이 혼재되어 있어 둘 다 처리해야 안전합니다.
- `fromTmFc`/`toTmFc`의 "미입력시 기본시각(00:00/23:59)" 안내는 일부 기능 문서에만 명시되어 있습니다.

## 7) 이 프로젝트에서 최소 준수해야 할 적용 포인트
